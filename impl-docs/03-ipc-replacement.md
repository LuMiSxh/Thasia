# IPC Replacement: Commands & Events

## Why IPC disappears

In Tauri, the frontend (JavaScript) and backend (Rust) are separate processes that
communicate through a serialized message channel. Every command crosses this bridge:
JS → serialize → IPC → deserialize → Rust → serialize → IPC → deserialize → JS.

In GPUI, **there is no frontend process**. The UI is Rust. Commands become plain
`async fn` calls. Events become `cx.notify()` on an entity. The IPC layer, all
Tauri Specta bindings, and all TypeScript types in `src/types/bindings.ts` are deleted.

---

## Command mapping

Each Tauri `#[tauri::command]` becomes a plain `async fn` (or sync fn) in a module.
The function signature loses `State<'_, ...>` extractors and `tauri::Window` — instead
it receives data directly or reads from `cx.global()` inside the calling entity.

### Scan commands

```rust
// Before (src-tauri/src/commands/scan.rs)
#[tauri::command]
pub async fn scan_source(
    path: String,
    state: State<'_, RwLock<ConvState>>,
) -> Result<Vec<VolumeMeta>, AppError> { ... }

// After (same file, stripped of Tauri)
pub async fn scan_source(path: PathBuf) -> Result<Vec<VolumeMeta>, AppError> { ... }
pub async fn scan_current_source(source: LocalSource) -> Result<Vec<VolumeMeta>, AppError> { ... }
```

**Called from GPUI:**

```rust
// Inside ConvertView action handler:
let path = self.source_path.clone().unwrap();
let handle = cx.entity_handle();
cx.spawn(|cx| async move {
    match scan_source(path).await {
        Ok(volumes) => handle.update(&mut cx, |view, cx| {
            view.scan_result = Some(volumes);
            view.step = WizardStep::Configure;
            cx.notify();
        }),
        Err(e) => handle.update(&mut cx, |view, cx| {
            view.error = Some(e);
            cx.notify();
        }),
    }
}).detach();
```

### Convert commands

```rust
// After
pub async fn build_pipeline_plan(
    source: &LocalSource,
    options: &ConvertOptions,
    edits: &[VolumeEdit],
) -> Result<PipelinePlan, AppError> { ... }

pub async fn convert(
    source: LocalSource,
```

The `on_progress` callback replaces Tauri events. See `05-async-and-events.md`.

## Event mapping

Tauri events were the only mechanism for the backend to push progress to the frontend.
In GPUI they become entity mutation + `cx.notify()`:

| Tauri event                            | GPUI equivalent                                                 |
| -------------------------------------- | --------------------------------------------------------------- |
| `ScanProgressEvent { current, total }` | `view.scan_progress = (current, total); cx.notify()`            |
| `VolumeStartEvent { .. }`              | `view.progress = Progress::Volume { .. }; cx.notify()`          |
| `ImageProgressEvent { .. }`            | `view.progress = Progress::Image { .. }; cx.notify()`           |
| `VolumeCompleteEvent { .. }`           | `view.completed_volumes.push(..); cx.notify()`                  |
| `ConversionCompleteEvent { .. }`       | `view.summary = Some(..); view.is_running = false; cx.notify()` |
| `SuwayomiStateChangedEvent { state }`  | `discover_view.runtime_state = state; cx.notify()`              |
| `SuwayomiInstallProgressEvent { .. }`  | `discover_view.install_progress = ..; cx.notify()`              |
| `DownloadStartEvent { .. }`            | `discover_view.download_progress = Some(..); cx.notify()`       |
| `ChapterDownloadEvent { .. }`          | `discover_view.download_progress = Some(..); cx.notify()`       |
| `DownloadCompleteEvent { .. }`         | `discover_view.download_progress = None; cx.notify()`           |

## Type definitions

All types currently in `src/types/bindings.ts` (auto-generated by tauri-specta) move to
the Rust crates they belong to and are referenced directly. No serialization layer needed.

| TypeScript type               | Rust home          |
| ----------------------------- | ------------------ |
| `VolumeMeta`                  | `thasia-core`      |
| `PipelinePlan`                | `thasia-core`      |
| `ConvertOptions`              | `thasia-core`      |
| `VolumeEdit`                  | `thasia-core`      |
| `SourceInfo`, `ExtensionInfo` | `thasia-source`    |
| `SearchPage`, `ChapterMeta`   | `thasia-source`    |
| `DiscoverySettings`           | `thasia-source`    |
| `RuntimeState`                | `thasia-source`    |
| `AppError`                    | `src/` (app crate) |
