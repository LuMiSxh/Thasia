# Thasia вЂ” Tauri UI Design Spec

**Date:** 2026-04-18  
**Status:** Approved for implementation

---

## Overview

Add a Tauri 2 desktop application to the Thasia workspace with a SvelteKit frontend. The UI is a wizard-based converter, adapted from Palaxy's UX, with a modular step registry, a context-aware collapsible sidebar, and a memory-efficient page editor that uses a custom URI protocol instead of passing image bytes through IPC.

No CSS styling in the initial implementation вЂ” raw HTML with positional styling only. Tailwind CSS v4 is wired up but left unstyled for now.

### Managed State (`state.rs`)

```rust
pub struct ConvState {
    pub scan_result: Option<Vec<(u32, Vec<ParsedImage>)>>,  // volume_num в†’ pages
    pub edited_pages: Option<Vec<(u32, Vec<PageEntry>)>>,   // after page editor
    pub options: ConvertOptions,
}

pub struct PageEntry {
    pub parsed: Option<ParsedImage>,  // None = user-added custom image
    pub custom_path: Option<PathBuf>, // set for user-added images
    pub excluded: bool,
}
```

Wrapped in `tokio::sync::Mutex<ConvState>` and registered via `.manage()`.

### Commands

**`scan_source(path: String) в†’ Result<Vec<(u32, Vec<PageMeta>)>>`**  
Runs `LocalSource::new` / `LocalSource::from_archive`, calls `source.discover()`, runs `Resolver::resolve` on each image, groups by volume number from `ChapterIdentifier`. Stores the full result in `ConvState`. Returns `Vec<(u32, Vec<PageMeta>)>` where `PageMeta` contains only the metadata JS needs: `{ volume_num, page_index, url: "thasia://image/{vol}/{idx}", file_name, is_cover }` вЂ” no image bytes.

**`convert(options: ConvertOptions) в†’ Result<()>`**  
Reads `ConvState.edited_pages`, filters excluded pages, runs `start_pipeline` в†’ packager for each volume. Emits progress events throughout.

**`update_pages(edits: Vec<(u32, Vec<PageEditAction>)>) в†’ Result<()>`**  
Applies the page editor's mutations (reorder, exclude, add custom) to `ConvState.edited_pages`.

### Custom Protocol (`protocol.rs`)

Registered as `thasia://image/{volume_num}/{page_index}`.

Uses `register_asynchronous_uri_scheme_protocol` so file I/O is non-blocking. The handler:
1. Parses `volume_num` and `page_index` from the URI path
2. Looks up the file path from `ConvState.scan_result` (already in managed state вЂ” no IPC)
3. Reads the file from disk (works for both directory sources and extracted TempDir from ZIP/CBZ)
4. Returns bytes with the correct MIME type (`image/jpeg`, `image/png`, etc.)

**Memory profile:** zero image bytes in the JS heap. Only the `src` URL strings are held in the component. The WebView's native image cache handles repeat requests.

Must be declared in `tauri.conf.json` capabilities:
```json
{ "protocol": { "asset": true, "assetScope": ["thasia://**"] } }
```

### Events (`events.rs`)

All derive `specta::Type` + `tauri_specta::Event`.

| Event | Payload |
|---|---|
| `ScanProgressEvent` | `{ current: u32, total: u32 }` |
| `VolumeStartEvent` | `{ volume_num: u32, volume_name: String, total_volumes: u32 }` |
| `ImageProgressEvent` | `{ volume_num: u32, current: u32, total: u32 }` |
| `VolumeCompleteEvent` | `{ volume_num: u32, success: bool, error: Option<String> }` |
| `ConversionCompleteEvent` | `{ successful: u32, failed: u32, duration_secs: f64 }` |

### Specta / Bindings

In debug builds, `tauri-specta` exports all command signatures and event types to `src/types/bindings.ts`. Pattern mirrors Palaxy exactly:

```rust
let specta_builder = SpectaBuilder::<tauri::Wry>::new()
    .commands(collect_commands![commands::scan::scan_source, commands::convert::convert, commands::convert::update_pages])
    .events(collect_events![events::ScanProgressEvent, /* ... */]);

#[cfg(debug_assertions)]
specta_builder.export(Typescript::default(), "../src/types/bindings.ts").unwrap();
```

## Frontend

### Toolchain

- SvelteKit 2 + Svelte 5 + `@sveltejs/adapter-static`
- Tailwind CSS v4 (`@tailwindcss/vite` plugin) вЂ” wired but no styles applied initially
- pnpm, Vite 7, TypeScript 5

### App Shell

`+layout.svelte` renders:
```
<Sidebar />
<slot />   в†ђ page content takes remaining space
```

The sidebar toggle button is always visible on the left edge, vertically centered. Sidebar width animates open/closed via a CSS class.

### Sidebar (`Sidebar.svelte` + `sidebar/state.svelte.ts`)

State:
```ts
let isOpen = $state(false)     // user preference, persists across navigation
let mode = $state<'nav' | 'wizard'>('nav')
```

Context overrides (called by the wizard host):
- Entering `/convert` в†’ `mode = 'wizard'`, `isOpen = true`  
- Step 9 (Convert) active в†’ `isOpen = false`  
- Leaving `/convert` в†’ `mode = 'nav'`

In `nav` mode: Home / Convert / Settings links.  
In `wizard` mode: step progress list. Each step shows:
- вњ“ green + clickable в†’ completed steps (jump back)
- в—Џ active в†’ current step  
- в—‹ solid в†’ locked (not yet reached)
- в—‹ dashed + italic в†’ conditional (will appear/disappear based on state)

### Wizard (`wizard/steps.ts` + `wizard/state.svelte.ts`)

Step type:
```ts
type WizardStep = {
  id: string
  label: string
  component: Component
  condition?: (state: WizardState) => boolean
}
```

Registry (array, edit to add/remove/reorder):

| # | id | label | condition |
|---|---|---|---|
| 1 | `source` | Source | вЂ” |
| 2 | `destination` | Destination | вЂ” |
| 3 | `image-format` | Image Format | вЂ” |
| 4 | `container` | Container | вЂ” |
| 5 | `direction` | Direction | `s.container === 'epub'` |
| 6 | `bundling` | Bundling | вЂ” |
| 7 | `page-editor` | Pages | вЂ” |
| 8 | `review` | Review | вЂ” |
| 9 | `convert` | Convert | вЂ” |

The wizard host in `convert/+page.svelte` filters the registry through conditions at render time. Steps where `condition(state) === false` are hidden from both the content area and the sidebar progress list (shown dashed if the condition hasn't resolved yet).

### Step 7 вЂ” Page Editor (`Step7PageEditor.svelte`)

Receives `PageMeta[]` per volume from Svelte state (populated after `scan_source` completes in Step 1).

- Left panel: volume list вЂ” click to switch active volume
- Main grid: image cards rendered as `<img src="{page.url}" />` where `url = "thasia://image/{vol}/{idx}"`
- **No image bytes in JS** вЂ” the WebView fetches each image via the custom protocol only when it's scrolled into view
- First card in each volume: purple COVER badge
- User-added images: green ADDED badge
- Excluded pages: visible, faded, dashed border вЂ” click to restore
- Drag-and-drop reorder (using native HTML5 drag events or a lightweight lib)
- "Add image" button: opens Tauri file dialog, inserts `PageEntry` with `custom_path` set
- On "Next": calls `update_pages(edits)` to write mutations back to Rust state

### Settings (`settings/+page.svelte`)

Form with default values for every wizard field. Saved to Tauri's app data dir (or localStorage for now). Pre-fills wizard state on wizard start.
